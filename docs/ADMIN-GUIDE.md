# دليل الإدارة — المكتبة الطيبة

هذا الدليل يشرح كيفية ضبط حسابات المسؤولين، إخفاء الكتب، وحجب الأقسام
المقيّدة عن الزوّار العاديّين.

> **مهمّ جداً قبل البدء:** الملف الجديد `data/admin-config.json` يحتوي على
> البريد الإلكتروني المؤقّت `REPLACE_WITH_OWNER_EMAIL@example.com`. يجب
> استبداله ببريدك الإلكتروني الفعلي قبل أن تعمل ميزات المسؤول.

---

## 1) إضافة بريد إلكتروني للمسؤول

عدّل `data/admin-config.json` وأضف عنوان بريدك إلى مصفوفة `adminEmails`:

```json
{
  "adminEmails": [
    "your-real-email@example.com",
    "co-admin@example.com"
  ]
}
```

عند الدخول للموقع بحساب Firebase يحمل هذا البريد، ستحصل تلقائياً على
صلاحيات المسؤول، ويتمّ تعيين `window.IS_ADMIN = true`.

**ما الذي يفعله المسؤول؟**
- يرى كل الكتب — حتى المخفيّة (مع وسم "محذوف" أحمر)
- يرى كل الأقسام — حتى المقيّدة
- يظهر له زر ✕ صغير في زاوية كل بطاقة كتاب لحذفه/إخفائه
- يظهر له شريط "إخفاء / إعادة الإظهار" في صفحة تفاصيل أي كتاب

---

## 2) إعطاء وصول لشخص ما إلى قسم "الإسلاميات"

في الإعدادات الافتراضية، قسم **الدين والإسلاميات** محجوب عن جميع
الزوّار. لإتاحته لأشخاص محدّدين دون منحهم صلاحيات إدارية كاملة، أضف
بريدهم إلى `categoryAllowedEmails`:

```json
{
  "categoryAllowedEmails": [
    "your-real-email@example.com",
    "scholar@example.com"
  ]
}
```

> ملاحظة: المسؤولون (الذين في `adminEmails`) يرون كل الأقسام تلقائياً —
> لا حاجة لإضافتهم هنا.

لإلغاء حجب القسم عن الجميع، احذف اسمه من `restrictedCategories`:

```json
{
  "restrictedCategories": []
}
```

---

## 3) إخفاء كتاب معيّن من كل الزوّار (إخفاء عالمي)

أضف معرف الكتاب (`id` من ملفات JSON) إلى مصفوفة `hiddenBooks`:

```json
{
  "hiddenBooks": ["218", "219", "220"]
}
```

هذا يخفي الكتب عن **كل** المستخدمين (باستثناء المسؤولين الذين يرونها
معتمة مع وسم "محذوف"). لا يتطلّب الأمر تعديل ملفات `books-extra-*.json`
ذاتها.

> بديل سريع: المسؤول يمكنه الضغط على × في بطاقة الكتاب فيُحفظ المعرف في
> `localStorage` الخاص بجهازه فقط (هذا إخفاء محلّي لا يصل بقيّة الزوّار).

---

## 4) إخفاء أحد الكتب من جهازك فقط (إخفاء محلّي)

عند تسجيل الدخول كمسؤول، اضغط زر × على أي بطاقة. يُحفظ المعرف في
`localStorage` تحت المفتاح `taybaa-hidden-books`. كذلك يُرسل طلب POST
إلى `/api/admin/hide` (حاليّاً يردّ 202 فقط — مرحلة KV لاحقاً).

لمزامنة الإخفاء عبر الأجهزة، نقل القائمة لاحقاً إلى `hiddenBooks` في
`data/admin-config.json` ثم اعمل commit.

---

## 5) الترقية إلى Cloudflare KV (مستقبلاً)

التعديل الحالي يكتفي بـ `localStorage` + commit يدوي لملف الإعدادات.
عند تجهيز KV namespace باسم `ADMIN_KV`:

1. عرّف الـbinding في `wrangler.toml`:
   ```toml
   [[kv_namespaces]]
   binding = "ADMIN_KV"
   id = "..."
   ```
2. عدّل `functions/api/admin/hide.js` ليقرأ ويكتب من
   `env.ADMIN_KV.get('hidden-books', 'json')`. التعليق داخل الملف يبيّن
   الكود تفصيلاً.
3. أزل الحاجة لتعديل `data/admin-config.json` يدوياً.

---

## 6) ملخّص الملفات المعدَّلة في هذه الموجة

- `data/admin-config.json` — ملف الإعدادات الجديد (عدّل البريد!)
- `js/admin-role.js` — منطق اكتشاف المسؤول + الفلترة
- `js/homepage-v2.js` — يستدعي الفلترة قبل عرض الكتب والأقسام
- `js/book.js` — يحجب صفحة الكتاب إذا كان مقيّداً أو مخفيّاً
- `category.html` — صفحة قسم مستقلة مع حماية الفئات المقيّدة
- `functions/api/admin/hide.js` — endpoint POST (stub حاليّاً)
- `book.html`, `v2.html` — تضمين `js/admin-role.js` وزر الإخفاء

> **تذكير أخير:** افتح `data/admin-config.json` الآن واستبدل
> `REPLACE_WITH_OWNER_EMAIL@example.com` ببريدك الفعلي، ثم اعمل commit
> وانتظر Cloudflare Pages أن يُعيد البناء (دقيقة تقريباً).
